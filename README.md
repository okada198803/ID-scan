# 📇 名刺スキャナー

> ブラウザだけで動く、完全無料の名刺データ化ツール  
> APIキー不要 / インストール不要 / 外部サーバーへの送信なし

---

## 📌 概要

名刺をカメラ撮影またはファイルアップロードで読み込み、OCR（文字認識）によって情報を自動抽出・分類し、Excel / CSV へ出力できる Web アプリです。単一の HTML ファイルで完結しており、ブラウザで開くだけで即使用できます。

---

## ✨ 主な機能

| 機能 | 詳細 |
|------|------|
| **画像入力** | ファイル選択 / ドラッグ＆ドロップ / カメラ撮影（スマートフォン対応） |
| **高精細撮影** | 最大 4096×2160 の高解像度でキャプチャ。カメラ画面に実解像度をリアルタイム表示 |
| **向き選択** | 横向き / 縦向きを切り替えてガイド枠・OCRモードを最適化 |
| **プリクロップ** | カメラガイド枠内の領域だけを撮影時に自動切り出し |
| **トリミング** | 読み込み後に名刺部分を精密切り出し（回転・比率固定対応） |
| **画像前処理** | 6段階パイプラインで OCR 精度を向上（詳細後述） |
| **OCR** | Tesseract.js（jpn / jpn_vert / eng 3モデル）による認識 |
| **スペース除去** | OCR 起因の文字間スペースを自動クリーニング |
| **自動分類** | 強化ルールベースエンジンで 7 種フィールドに仕分け |
| **手動修正** | 分類結果を保存前に各フィールドで直接編集可能 |
| **一覧管理** | 登録済み名刺の検索・削除・OCR 原文表示 |
| **出力** | Excel（.xlsx）/ CSV（.csv）/ 印刷 の3形式 |

## 🚀 使い方

```
1. meishi_scanner.html をダウンロードしてブラウザで開く
2. 名刺の向き（横 / 縦）を選択する
3. 名刺画像を読み込む（ファイル選択 or カメラ撮影）
4. トリミングで名刺部分だけを切り出す
5. 「トリミングして OCR へ」をクリック
6. 分類結果を確認・必要なら修正する
7. 「保存する」で名刺リストに追加
8. 右パネルから Excel / CSV へエクスポート
```

### ステップ詳細

#### ① 向きを選ぶ
カメラ起動前に **横向き（デフォルト）** または **縦向き** を選択します。選択に応じてカメラのガイド枠のアスペクト比・OCR モード・トリマーの比率が自動で切り替わります。

| 設定 | ガイド枠比率 | OCR モデル | ページ解析 |
|------|------------|-----------|-----------|
| 横向き | 1.585（85.6×54mm） | jpn + eng | PSM 3（全自動） |
| 縦向き | 0.631（54×85.6mm） | jpn_vert + jpn + eng | PSM 5（縦書きブロック） |

#### ② 名刺を読み込む
- **ファイル選択**：JPG / PNG / WebP に対応
- **ドラッグ＆ドロップ**：ブラウザ画面に直接ドロップ
- **カメラ撮影**：最大 4096×2160 の高解像度でキャプチャ。ガイド枠内側だけを撮影と同時にプリクロップしてトリマーへ渡します

#### ③ トリミング
撮影・読み込み直後にトリミング画面が表示されます。アスペクト比は選択した向きに固定（「フリー」ボタンで解除可能）。

| 操作 | 説明 |
|------|------|
| ドラッグ | 切り出し範囲を移動 |
| 左 / 右回転ボタン | 90° 単位で回転 |
| リセット | 元の状態に戻す |
| フリー | アスペクト比を解除して自由に選択 |

#### ④ 分類結果を確認・修正
自動分類されたテキストが 7 種のフィールドに表示されます。各フィールドは直接編集でき、複数値は改行で区切って入力できます。「OCR 生テキストを確認・再分類」から原文を編集して再分類することも可能です。

## 🖼️ 画像前処理パイプライン

OCR に渡す前に以下の 6 段階処理を自動実行します。

| ステップ | 処理内容 | 効果 |
|---------|---------|------|
| 1 | **アップスケール**（短辺 2400px 未満は最大 2.5倍） | 低解像度画像の文字を拡大 |
| 2 | **グレースケール変換**（輝度ベース加重） | カラー情報除去でノイズ低減 |
| 3 | **ヒストグラム伸長**（1〜99 パーセンタイル正規化） | コントラスト自動補正 |
| 4 | **アンシャープマスク**（ガウスぼかし差分） | 文字エッジの鮮明化 |
| 5 | **Sauvola 適応的二値化**（インテグラルイメージ O(n)） | 照明ムラ・グラデーション背景に強い局所閾値処理 |
| 6 | **縦書き時 90°回転**（縦書きを横書きとして読む） | jpn_vert の弱点を横書きモデルで補完 |

進捗バーは「前処理 10% → OCR 85% → 完了 100%」の合算で表示されます。

## 🔧 スペース自動除去

Tesseract.js は文字間に不要なスペースを挿入することがあります。以下のクリーニングを自動実行します。

```
株 式 会 社          →   株式会社        （漢字間スペース除去）
山 田 太 郎          →   山田太郎        （漢字間スペース除去）
user @ mail . jp     →   user@mail.jp    （メールアドレス正規化）
03 - 1234 - 5678     →   03-1234-5678    （電話番号正規化）
〒 123 - 4567        →   〒123-4567      （郵便番号正規化）
```

処理後のテキストは「OCR 生テキスト」欄で確認・手動修正できます。

## 🔒 プライバシー・セキュリティ

- **すべての処理はブラウザ内で完結**します
- 画像・テキスト・名刺データは**外部サーバーへ一切送信されません**
- 名刺データは**ブラウザの `localStorage`** に保存されます
- API キーやアカウント登録は**不要**です

> ブラウザのデータを消去するとデータが失われます。定期的に Excel または CSV でエクスポートしてバックアップしてください。

## ❓ よくある質問

**Q. OCR の認識精度が低い場合は？**  
A. トリミングで名刺部分だけを切り出すと精度が改善します。また「OCR 生テキスト」欄を直接編集して「再分類」ボタンで再処理できます。

**Q. 縦書き名刺の精度が悪い場合は？**  
A. 向き選択で「縦向き」を選ぶと `jpn_vert` モデルが追加ロードされ、前処理で画像を 90° 回転してから横書き OCR もかけます。それでも精度が低い場合は OCR 生テキストを手動修正してください。

**Q. TEL や電話という文字が番号と一緒に入ってしまう**  
A. 接頭辞（TEL: / 電話: 等）は自動除去されます。括弧形式の市外局番も正規化されます。

**Q. 英語名が大文字だけで認識された**  
A. `JOHN SMITH` のような全大文字の英語名は自動で `John Smith` に正規化されます。

**Q. データはどこに保存されますか？**  
A. ブラウザの `localStorage` に保存されます。別のブラウザやデバイスからはアクセスできません。定期的に CSV または Excel でエクスポートすることをお勧めします。

**Q. 一度に複数の名刺を登録できますか？**  
A. 1枚ずつ処理する仕様です。登録後は同じ操作を繰り返して複数の名刺を蓄積できます。
