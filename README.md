# 📇 名刺スキャナー

> ブラウザだけで動く、完全無料の名刺データ化ツール  
> APIキー不要 / インストール不要 / 外部サーバーへの送信なし

---

## 📌 概要

名刺をカメラ撮影またはファイルアップロードで読み込み、OCR（文字認識）によって情報を自動抽出・分類し、Excel / CSV / HTML へ出力できる Web アプリです。単一の HTML ファイルで完結しており、ブラウザで開くだけで即使用できます。

---

## ✨ 主な機能

| 機能 | 詳細 |
|------|------|
| **画像入力** | ファイル選択 / ドラッグ＆ドロップ / カメラ撮影（スマートフォン対応） |
| **高精細撮影** | 最大 4096×2160 の高解像度でキャプチャ。カメラ画面に実解像度をリアルタイム表示 |
| **向き選択** | 横向き / 縦向きを切り替えてガイド枠・OCRモードを最適化 |
| **プリクロップ** | カメラガイド枠内の領域だけを撮影時に自動切り出し |
| **トリミング** | 読み込み後に名刺部分を精密切り出し（回転・比率固定・フリー対応） |
| **画像前処理** | 7段階パイプラインで OCR 精度を向上（白抜き文字対応含む） |
| **OCR** | Tesseract.js（jpn / jpn_vert / eng 3モデル）による認識 |
| **スペース除去** | OCR 起因の文字間スペースを自動クリーニング |
| **OCR誤認識補正** | 都道府県・カタカナ・漢字の形状類似誤読を自動補正 |
| **自動分類** | 4パス・位置文脈対応ルールベースエンジンで 7 種フィールドに仕分け |
| **電話種別分類** | 固定電話（通話）/ FAX / 携帯番号を個別フィールドに分離 |
| **姓名辞書** | 日本語苗字300種・英語姓163種・欧州姓178種による氏名判定強化 |
| **手動修正** | 分類結果を保存前に各フィールドで直接編集可能 |
| **タグ検索** | 「その他」フィールドにタグを登録しワンタップでフィルタリング |
| **一覧管理** | 登録済み名刺の全文検索・削除・OCR 原文表示 |
| **出力** | Excel（.xlsx）/ CSV（.csv）/ HTML（タグ検索付き）/ 印刷 の4形式 |

## 🚀 使い方

```
1. meishi_scanner.html をダウンロードしてブラウザで開く
2. 名刺の向き（横 / 縦）を選択する
3. 名刺画像を読み込む（ファイル選択 or カメラ撮影）
4. トリミングで名刺部分だけを切り出す
5. 「トリミングして OCR へ」をクリック
6. 分類結果を確認・必要なら修正する
7. 「保存する」で名刺リストに追加
8. 右パネルから Excel / CSV / HTML へエクスポート
```

### 向きを選ぶ

| 設定 | ガイド枠比率 | OCR モデル | ページ解析 |
|------|------------|-----------|-----------|
| 横向き（デフォルト） | 1.585（85.6×54mm） | jpn + eng | PSM 3（全自動） |
| 縦向き | 0.631（54×85.6mm） | jpn_vert + jpn + eng | PSM 5（縦書きブロック） |

### トリミング操作

| 操作 | 説明 |
|------|------|
| ドラッグ | 切り出し範囲を移動 |
| 左 / 右回転ボタン | 90° 単位で回転 |
| リセット | 元の状態に戻す |
| フリー | アスペクト比を解除して自由に選択 |

## 🖼️ 画像前処理パイプライン（7段階）

OCR に渡す前に以下の処理を自動実行します。

| ステップ | 処理内容 | 効果 |
|---------|---------|------|
| 1 | **アップスケール**（短辺 2400px 未満は最大 2.5倍） | 低解像度画像の文字を拡大 |
| 2 | **グレースケール変換**（輝度ベース加重） | カラー情報除去でノイズ低減 |
| 2.5 | **白抜き文字検出・反転** | 暗い背景の白文字を自動検出して反転（詳細後述） |
| 3 | **ヒストグラム伸長**（1〜99 パーセンタイル正規化） | コントラスト自動補正 |
| 4 | **アンシャープマスク**（ガウスぼかし差分） | 文字エッジの鮮明化 |
| 5 | **Sauvola 適応的二値化**（インテグラルイメージ O(n)） | 照明ムラ・グラデーション背景に強い局所閾値処理 |
| 6 | **縦書き時 90°回転** | jpn_vert の弱点を横書きモデルで補完 |

## ⬛ 白抜き文字対応

黒・紺・グレーなどの暗い背景に白文字で印刷された名刺は、Tesseract.js がそのままでは認識できません。本アプリは以下の方式で自動対応します。

### 検出ロジック

画像を水平方向に **6分割（スライス）** し、各スライスの暗さを個別に判定します。

```
┌─────────────────────────────┐  ← スライス1: 暗い（55%超が暗い） → 反転
│  ████ 株式会社〇〇 ████    │
├─────────────────────────────┤  ← スライス2: 暗い → 反転
│  ████ 山田 太郎 ████       │
├─────────────────────────────┤  ← スライス3: 明るい → そのまま
│  TEL: 03-1234-5678         │
├─────────────────────────────┤  ← スライス4: 明るい → そのまま
│  Email: yamada@example.com  │
└─────────────────────────────┘
```

| 判定 | 条件 | 処理 |
|------|------|------|
| **全体反転** | 画像全体の55%超が暗い | 全ピクセルを一括反転 |
| **部分反転** | スライス単位で50%超が暗い | そのスライスのみ反転 |
| **処理なし** | 明るい背景 | 従来の前処理のみ |

### 対応できる名刺の例

- 黒背景全面に白文字
- 上部ヘッダーバンドが紺色・白文字、下部が白地・黒文字
- グラデーション背景（暗→明）に白文字

## 🔧 OCR誤認識補正

OCR 後に以下の補正テーブルを自動適用します。

### 文字形状類似誤認識

| 誤読パターン | 補正後 | 原因 |
|------------|--------|------|
| `人阪` / `人阪府` | `大阪` / `大阪府` | 「大」の上部が欠けて「人」に化ける |
| `人学` / `人手` / `人臣` | `大学` / `大手` / `大臣` | 同上 |
| `ちあさき` | `ちあき` | OCRがストロークを「さ」と誤挿入 |
| `アカラーコジサルタンジ` | `カラーコンサルタント` | 行頭余分な「ア」＋「ン↔ジ」置換 |
| `コンサルタンジ` | `コンサルタント` | 語末「ト→ジ」置換 |
| `弁護土` / `税理土` / `建築土` | `弁護士` / `税理士` / `建築士` | 「士」↔「土」の混同 |

### 都道府県の誤認識補正

東京都 / 大阪府 / 神奈川県 / 愛知県 / 埼玉県 / 千葉県 / 福岡県 / 広島県 / 京都府 / 兵庫県 / 北海道 の誤認識パターンを個別に補正します。

### その他補正対象

法人格（株式会社・有限会社等）/ メールアドレス（全角@・ドメイン誤字）/ 電話番号（全角数字・接頭辞）/ 英語法人格（Inc. / Ltd. / Corp.）/ 住所（丁目・番地・郵便番号）

## 🧠 自動分類エンジン（4パス）

OCR テキストを以下の順で処理し、7種のフィールドに振り分けます。

```
PASS 1: 強ルール × 位置ボーナス × 文脈ボーナス × 除外フィルター
  ↓ 未分類の行のみ
PASS 2: 弱ルール × 同上
  ↓ 未分類の行のみ
PASS 3: フォールバック再分類
  （電話番号再試行 / 氏名再試行 / 部署名→役職 / URL分離）
  ↓ 残り
PASS 4: ノイズ除去 → other
```

### 位置・文脈判定

名刺の典型的なレイアウト（上部＝会社名、中部＝氏名・役職、下部＝連絡先）を利用してスコアを加減算します。

| フィールド | 高スコア位置 | 文脈加算条件 |
|-----------|------------|------------|
| 会社名 | 上部 35% 以内 | — |
| 氏名 | 中部 20〜65% | 直前に会社名 +2 / 直後に役職 +2 |
| 役職 | 中部 25〜70% | 直前が氏名候補 +2 |
| 住所 | 下部 50% 以降 | 直前が郵便番号行 +3 |

### 複数行マージ

| 対象 | 内容 |
|------|------|
| 住所 | 郵便番号行を起点に最大3行を自動結合 |
| 会社名 | 社名が2行に分かれている場合に直後1行を結合 |

## ♻️ 継続的な運用フロー

複数回に分けて名刺を蓄積していく場合の推奨フローです。

```
【初回】
  名刺スキャン → 保存 → CSV / Excel / HTML 出力

【2回目以降】
  ① 前回の CSV / Excel をアプリの「過去データの取り込み」から読み込む

  ② 新しい名刺をスキャン・追加
  ③ 更新された CSV / Excel / HTML を出力

【閲覧・共有】
  HTML ファイル → ブラウザで開くだけでタグ検索が使える
  Excel ファイル → オートフィルターでタグ列を検索
```

- **すべての処理はブラウザ内で完結**します

## ❓ よくある質問

**Q. 白抜き文字の名刺が読み取れない**  
A. 前処理の Step 2.5 で自動検出・反転します。全体が暗い名刺は一括反転、上部バンドのみ暗い名刺はその帯だけを反転します。それでも精度が低い場合は OCR 生テキスト欄を手動修正してください。

**Q. OCR の認識精度が低い場合は？**  
A. トリミングで名刺部分だけを切り出すと精度が改善します。また「OCR 生テキスト」欄を直接編集して「再分類」ボタンで再処理できます。

**Q. 縦書き名刺の精度が悪い場合は？**  
A. 向き選択で「縦向き」を選ぶと `jpn_vert` モデルが追加ロードされ、前処理で 90° 回転してから横書き OCR もかけます。

**Q. 「大阪」が「人阪」と読まれる**  
A. OCR 誤認識補正テーブルで自動修正されます。「人阪」「大阪」どちらで認識されても「大阪」に変換されます。

**Q. 欧州系の名前が分類されない**  
A. 欧州圏の姓辞書（178種）を収録しており、ドイツ・フランス・イタリア・北欧・東欧等の姓名を氏名として認識します。

**Q. タグ検索は CSV / Excel でも使えますか？**  
A. CSV / Excel の「タグ・その他」列でテキストフィルターを使って検索できます。より手軽に使うには HTML 形式でエクスポートするとタグクリック検索UIが使えます。

**Q. データはどこに保存されますか？**  
A. ブラウザの `localStorage` に保存されます。定期的に CSV または Excel でエクスポートすることをお勧めします。
