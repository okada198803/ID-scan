# 📇 名刺スキャナー

> ブラウザだけで動く、完全無料の名刺データ化ツール  
> APIキー不要 / インストール不要 / 外部サーバーへの送信なし

---

## 📌 概要

名刺をカメラ撮影またはファイルアップロードで読み込み、OCR（文字認識）によって情報を自動抽出・分類し、Excel / CSV へ出力できる Web アプリです。単一の HTML ファイルで完結しており、ブラウザで開くだけで即使用できます。

---

## ✨ 主な機能

| 機能 | 詳細 |
|------|------|
| **画像入力** | ファイル選択 / ドラッグ＆ドロップ / カメラ撮影（スマートフォン対応） |
| **高精細撮影** | 最大 4096×2160 の高解像度でキャプチャ。カメラ画面に実解像度をリアルタイム表示 |
| **向き選択** | 横向き / 縦向きを切り替えてガイド枠・前処理を最適化 |
| **プリクロップ** | カメラガイド枠内の領域だけを撮影時に自動切り出し |
| **トリミング** | 読み込み後に名刺部分を精密切り出し（回転・比率固定・フリー対応） |
| **画像前処理** | 7段階パイプラインで OCR 精度を向上（白抜き文字・カラー背景対応含む） |
| **OCR** | Tesseract.js（jpn + eng 2モデル）による認識。縦書き名刺は前処理で90°回転して横書きOCR |
| **スペース除去** | OCR 起因の文字間スペースを自動クリーニング |
| **OCR誤認識補正** | 都道府県・カタカナ・漢字・アイコン接頭辞・漢数字等を自動補正 |
| **自動分類** | 4パス・文脈対応ルールベースエンジンで 7 種フィールドに仕分け |
| **同一行分離** | 「役職＋氏名」「日本語名＋英語名」が同一行の名刺を自動分割 |
| **電話種別分類** | 固定電話 / FAX / 携帯番号を個別フィールドに分離。同一市外局番ルールで FAX を自動判定 |
| **防衛企業辞書** | 日本・海外の防衛関連企業 1,000 社超を収録。法人格がなくても会社名を認識 |
| **姓名辞書** | 日本語苗字 450種・英語姓 163種・欧州姓 178種による氏名判定強化 |
| **役職辞書** | 約 420種（防衛・商社・海外防衛産業・自衛隊英語略称含む） |
| **手動修正** | 分類結果を保存前に各フィールドで直接編集可能 |
| **タグ検索** | 「その他」フィールドにタグを登録しワンタップでフィルタリング |
| **一覧管理** | 登録済み名刺の全文検索・削除・OCR 原文表示 |
| **出力** | Excel（.xlsx）/ CSV（.csv）/ 印刷 の3形式 |

## 🚀 使い方

```
1. meishi_scanner.html をダウンロードしてブラウザで開く
2. 名刺の向き（横 / 縦）を選択する
3. 名刺画像を読み込む（ファイル選択 or カメラ撮影）
4. トリミングで名刺部分だけを切り出す
5. 「トリミングして OCR へ」をクリック
6. 分類結果を確認・必要なら修正する
7. 「保存する」で名刺リストに追加
8. 右パネルから Excel / CSV へエクスポート
```

### 向きを選ぶ

| 設定 | ガイド枠比率 | OCR 処理 |
|------|------------|---------|
| 横向き（デフォルト） | 1.585（85.6×54mm） | jpn + eng で直接認識 |
| 縦向き | 0.631（54×85.6mm） | 前処理で90°回転 → jpn + eng で横書き認識 |

### トリミング操作

| 操作 | 説明 |
|------|------|
| ドラッグ | 切り出し範囲を移動 |
| 左 / 右回転ボタン | 90° 単位で回転 |
| リセット | 元の状態に戻す |
| フリー | アスペクト比を解除して自由に選択 |

## 🖼️ 画像前処理パイプライン（7段階）

| ステップ | 処理内容 | 効果 |
|---------|---------|------|
| 1 | **アップスケール**（短辺 2400px 未満は最大 2.5倍） | 低解像度画像の文字を拡大 |
| 2 | **グレースケール変換**（輝度ベース加重） | カラー情報除去でノイズ低減 |
| 2.5 | **白抜き文字検出・反転** | 暗い背景の白文字を自動検出して反転 |
| 3 | **ヒストグラム伸長**（1〜99 パーセンタイル正規化） | コントラスト自動補正 |
| 3.5 | **背景複雑度検出**（イラスト・カラー背景判定） | 模様背景の場合にメディアンフィルターと適応 Sauvola を切り替え |
| 4 | **アンシャープマスク** or **メディアンフィルター** | 背景の複雑さに応じて処理を切り替え |
| 5 | **Sauvola 適応的二値化**（ウィンドウ幅可変） | 照明ムラ・グラデーション背景に強い局所閾値処理 |
| 6 | **縦書き時 90°回転** | 縦書き名刺を横書きとして OCR にかける |

### 背景の複雑さによる処理切り替え

| 背景 | メディアンフィルター | アンシャープマスク強度 | Sauvola ウィンドウ |
|------|------|------|------|
| シンプル（白・単色） | なし | 1.4（強め） | R=16 |
| 複雑（イラスト・水彩・模様） | 3×3 適用 | 1.1（抑制） | R=32（広め） |

## ⬛ 白抜き文字対応

画像を水平方向に **6分割**し、各スライスの暗さを個別に判定します。

| 判定 | 条件 | 処理 |
|------|------|------|
| **全体反転** | 画像全体の 55% 超が暗い | 全ピクセルを一括反転 |
| **部分反転** | スライス単位で 50% 超が暗い | そのスライスのみ反転 |
| **処理なし** | 明るい背景 | 従来の前処理のみ |

## 🧠 自動分類エンジン（4パス）

```
PASS 1: 強ルール × 文脈ボーナス × 除外フィルター
  ↓ 未分類の行のみ
PASS 2: 弱ルール × 同上
  ↓ 未分類の行のみ
PASS 3: フォールバック再分類
  （役職+氏名同一行分離 / 日英名同一行分離 /
    電話番号再試行 / 氏名再試行 / 部署名→役職 / URL分離 / 認証ノイズ除去）
  ↓ 残り
PASS 4: ノイズ除去 → other
```

**位置スコアは使用しない：** 名刺ごとにレイアウトが異なるため、行の位置による重み付けは廃止しています。代わりに前後行の内容（直前に会社名・直後に役職など）を文脈ボーナスとして利用します。

### 同一行の自動分離

| パターン | 例 | 処理 |
|---------|-----|------|
| **役職＋氏名** | `マネージャー　秋山和輝` | 役職と氏名に分割 |
| **TEL/FAX 同一行** | `TEL:03-1111-2222　FAX:03-1111-3333` | 分割して各種別フィールドへ |
| **日本語名＋英語名** | `海野花子 Hanako Umino` | 日本語名を氏名フィールドへ |

## 📞 電話番号の種別分類

| フィールド | アイコン | 判定基準 |
|-----------|---------|---------|
| 固定電話（通話） | 📞 | `0X-XXXX-XXXX` 形式 |
| FAX | 🖨 | `FAX` / `Fax.` 接頭辞、または同一市外局番ルール（後述） |
| 携帯番号 | 📱 | `090` / `080` / `070` / `050` 先頭、または `Mobile:` 接頭辞 |

**同一市外局番ルール：** FAX 番号が明示されていない場合、固定電話が2本以上あり同一市外局番であれば、2本目以降を FAX として自動分類します（名刺の一般的な記載順序：固定→FAX を前提）。

**対応する接頭辞・形式：** `TEL:` / `Tel.` / `FAX:` / `Fax.` / `PHONE:` / `電話` / `直通` / `携帯` / `Mobile` / `内線` / ⓣⓕⓜ（丸囲みアイコン）/ `(03)1234-5678`（括弧形式）/ `045 306-8208`（スペース区切り）/ `+81`（国際プレフィックス）

## ❓ よくある質問

**Q. OCR が動かない / エラーになる**  
A. ブラウザのコンソールにエラーが表示されている場合、`file://` で開いていると CORS 制限で Tesseract.js が起動できないことがあります。ローカルサーバーを立てるか、ブラウザの拡張機能や設定で CORS を緩和してください。Chrome では `--allow-file-access-from-files` フラグが有効です。

**Q. 白抜き文字の名刺が読み取れない**  
A. 前処理の Step 2.5 で自動検出・反転します。全体が暗い名刺は一括反転、上部バンドのみ暗い名刺はその帯だけを反転します。

**Q. カラー・イラスト背景の名刺の精度が低い**  
A. Step 3.5 で背景の複雑さを自動検出し、メディアンフィルターと広いウィンドウの Sauvola 二値化に切り替えます。それでも改善しない場合はトリミングで文字エリアを絞り込んでください。

**Q. 縦書き名刺の精度が悪い**  
A. 向き選択で「縦向き」を選ぶと前処理で 90° 回転してから横書き OCR にかけます。

**Q. ⓣⓕⓜ などのアイコン付き名刺が正しく分類されない**  
A. 丸囲み文字のアイコン接頭辞は `TEL:` / `FAX:` / `Mobile:` として自動認識されます。

**Q. 「マネージャー　秋山和輝」のように役職と名前が同じ行にある**  
A. PASS 3 の同一行分離処理が自動的に役職と氏名に分割します。

**Q. TEL と FAX が同じ行にある（`TEL:03-0000 FAX:03-0001`）**  
A. 分類前に自動で行分割してから処理します。

**Q. FAX 番号が明記されていない名刺**  
A. 固定電話が2本以上あり同一市外局番であれば、2本目を FAX として自動判定します。

**Q. 「丸紅エアロスペース」など法人格のない企業名が会社名として認識されない**  
A. 防衛関連企業辞書（1,000社超）に収録されており、法人格がなくても自動認識されます。

**Q. データはどこに保存されますか？**  
A. ブラウザの `localStorage` に保存されます。定期的に CSV または Excel でエクスポートすることをお勧めします。
