# 📇 名刺スキャナー

> ブラウザだけで動く、完全無料の名刺データ化ツール  
> APIキー不要 / インストール不要 / 外部サーバーへの送信なし

---

## 📌 概要

名刺をカメラ撮影またはファイルアップロードで読み込み、OCR（文字認識）によって情報を自動抽出・分類し、Excel / CSV / HTML へ出力できる Web アプリです。単一の HTML ファイルで完結しており、ブラウザで開くだけで即使用できます。

---

## ✨ 主な機能

| 機能 | 詳細 |
|------|------|
| **画像入力** | ファイル選択 / ドラッグ＆ドロップ / カメラ撮影（スマートフォン対応） |
| **高精細撮影** | 最大 4096×2160 の高解像度でキャプチャ。カメラ画面に実解像度をリアルタイム表示 |
| **向き選択** | 横向き / 縦向きを切り替えてガイド枠・OCRモードを最適化 |
| **プリクロップ** | カメラガイド枠内の領域だけを撮影時に自動切り出し |
| **トリミング** | 読み込み後に名刺部分を精密切り出し（回転・比率固定・フリー対応） |
| **画像前処理** | 7段階パイプラインで OCR 精度を向上（白抜き文字対応含む） |
| **OCR** | Tesseract.js（jpn / jpn_vert / eng 3モデル）による認識 |
| **スペース除去** | OCR 起因の文字間スペースを自動クリーニング |
| **OCR誤認識補正** | 都道府県・カタカナ・漢字の形状類似誤読・漢数字・アイコン接頭辞を自動補正 |
| **自動分類** | 4パス・位置文脈対応ルールベースエンジンで 7 種フィールドに仕分け |
| **同一行分離** | 「役職＋氏名」「日本語名＋英語名」が同一行に記載された名刺を自動分割 |
| **電話種別分類** | 固定電話（通話）/ FAX / 携帯番号を個別フィールドに分離 |
| **姓名辞書** | 日本語苗字300種・英語姓163種・欧州姓178種による氏名判定強化 |
| **手動修正** | 分類結果を保存前に各フィールドで直接編集可能 |
| **タグ検索** | 「その他」フィールドにタグを登録しワンタップでフィルタリング |
| **一覧管理** | 登録済み名刺の全文検索・削除・OCR 原文表示 |
| **出力** | Excel（.xlsx）/ CSV（.csv）/ HTML（タグ検索付き）/ 印刷 の4形式 |

## 🚀 使い方

```
1. meishi_scanner.html をダウンロードしてブラウザで開く
2. 名刺の向き（横 / 縦）を選択する
3. 名刺画像を読み込む（ファイル選択 or カメラ撮影）
4. トリミングで名刺部分だけを切り出す
5. 「トリミングして OCR へ」をクリック
6. 分類結果を確認・必要なら修正する
7. 「保存する」で名刺リストに追加
8. 右パネルから Excel / CSV / HTML へエクスポート
```

### 向きを選ぶ

| 設定 | ガイド枠比率 | OCR モデル | ページ解析 |
|------|------------|-----------|-----------|
| 横向き（デフォルト） | 1.585（85.6×54mm） | jpn + eng | PSM 3（全自動） |
| 縦向き | 0.631（54×85.6mm） | jpn_vert + jpn + eng | PSM 5（縦書きブロック） |

### トリミング操作

| 操作 | 説明 |
|------|------|
| ドラッグ | 切り出し範囲を移動 |
| 左 / 右回転ボタン | 90° 単位で回転 |
| リセット | 元の状態に戻す |
| フリー | アスペクト比を解除して自由に選択 |

## 🖼️ 画像前処理パイプライン（7段階）

| ステップ | 処理内容 | 効果 |
|---------|---------|------|
| 1 | **アップスケール**（短辺 2400px 未満は最大 2.5倍） | 低解像度画像の文字を拡大 |
| 2 | **グレースケール変換**（輝度ベース加重） | カラー情報除去でノイズ低減 |
| 2.5 | **白抜き文字検出・反転** | 暗い背景の白文字を自動検出して反転（詳細後述） |
| 3 | **ヒストグラム伸長**（1〜99 パーセンタイル正規化） | コントラスト自動補正 |
| 4 | **アンシャープマスク**（ガウスぼかし差分） | 文字エッジの鮮明化 |
| 5 | **Sauvola 適応的二値化**（インテグラルイメージ O(n)） | 照明ムラ・グラデーション背景に強い局所閾値処理 |
| 6 | **縦書き時 90°回転** | jpn_vert の弱点を横書きモデルで補完 |

## ⬛ 白抜き文字対応

黒・紺・グレーなどの暗い背景に白文字で印刷された名刺は、Tesseract.js がそのままでは認識できません。本アプリは以下の方式で自動対応します。

画像を水平方向に **6分割（スライス）** し、各スライスの暗さを個別に判定します。

```
┌─────────────────────────────┐  ← スライス1: 暗い（55%超）→ 反転
│  ████ 株式会社〇〇 ████    │
├─────────────────────────────┤  ← スライス2: 暗い → 反転
│  ████ 山田 太郎 ████       │
├─────────────────────────────┤  ← スライス3: 明るい → そのまま
│  TEL: 03-1234-5678         │
└─────────────────────────────┘
```

| 判定 | 条件 | 処理 |
|------|------|------|
| **全体反転** | 画像全体の 55% 超が暗い | 全ピクセルを一括反転 |
| **部分反転** | スライス単位で 50% 超が暗い | そのスライスのみ反転 |
| **処理なし** | 明るい背景 | 従来の前処理のみ |

## 🔧 OCR誤認識補正

OCR 後に以下の補正テーブルを自動適用します。

### 文字形状類似誤認識

| 誤読 | 補正後 | 原因 |
|------|--------|------|
| `人阪` / `人阪府` | `大阪` / `大阪府` | 「大」の上部欠損 |
| `ちあさき` | `ちあき` | OCRがストロークを「さ」と誤挿入 |
| `アカラーコジサルタンジ` | `カラーコンサルタント` | 行頭余分「ア」＋「ン↔ジ」置換 |
| `弁護土` / `税理土` / `建築土` | `弁護士` / `税理士` / `建築士` | 「士」↔「土」の混同 |

### 都道府県の誤認識補正

東京都 / 大阪府 / 神奈川県 / 愛知県 / 埼玉県 / 千葉県 / 福岡県 / 広島県 / 京都府 / 兵庫県 / 北海道

### 名刺サンプルから追加した補正

| パターン | 補正内容 | 対応する名刺の例 |
|---------|---------|--------------|
| ⓣⓕⓜⓔⓤ（丸囲みアイコン） | `TEL:` / `FAX:` / `Mobile:` / `E-mail:` / `URL:` に正規化 | アイコン付き名刺 |
| `Tel.` / `Fax.`（ドット形式） | `TEL:` / `FAX:` に正規化 | 欧米スタイル名刺 |
| `E-mail.` / `E-mail ` | `E-mail:` に正規化 | 多様な表記の名刺 |
| `PHONE:` / `E-MAIL:` | `TEL:` / `E-mail:` に正規化 | 英語表記名刺 |
| `045 306-8208`（スペース区切り） | `045-306-8208` にハイフン変換 | 市外局番スペース区切り |
| `(ext.2342)` | `内線2342` に変換 | 内線番号付き名刺 |
| `六丁目` / `五丁目` 等（漢数字） | `6丁目` / `5丁目` 等に変換 | 住所漢数字表記 |
| `(代表)` / `(直通)` 注記 | 電話番号末尾から除去 | 代表番号明記の名刺 |
| `認証:JIS Q...` / `ISO 9001` 等 | ノイズとして除去 | 認証番号記載の名刺 |
| `Tcertification` / `REGISTERED FIRM` | ノイズとして除去 | 認証マーク付き名刺 |

## 🧠 自動分類エンジン（4パス）

```
PASS 1: 強ルール × 位置ボーナス × 文脈ボーナス × 除外フィルター
  ↓ 未分類の行のみ
PASS 2: 弱ルール × 同上
  ↓ 未分類の行のみ
PASS 3: フォールバック再分類
  （役職+氏名同一行分離 / 日英名同一行分離 /
    電話番号再試行 / 氏名再試行 / 部署名→役職 / URL分離 / 認証ノイズ除去）
  ↓ 残り
PASS 4: ノイズ除去 → other
```

### 同一行の自動分離（PASS 3）

名刺によっては複数の情報が1行にまとまっている場合があります。

| パターン | 例 | 処理 |
|---------|-----|------|
| **役職＋氏名** | `マネージャー　秋山和輝` | 役職と氏名に分割して各フィールドへ |
| **日本語名＋英語名** | `海野花子 Hanako Umino` | 日本語名を氏名フィールドへ（英語名はother） |
| **英語名＋日本語名** | `Hanako Umino 海野花子` | 日本語名を氏名フィールドへ |

### 位置・文脈判定

| フィールド | 高スコア位置 | 文脈加算条件 |
|-----------|------------|------------|
| 会社名 | 上部 35% 以内 | — |
| 氏名 | 中部 20〜65% | 直前に会社名 +2 / 直後に役職 +2 |
| 役職 | 中部 25〜70% | 直前が氏名候補 +2 |
| 住所 | 下部 50% 以降 | 直前が郵便番号行 +3 |

### 複数行マージ

| 対象 | 内容 |
|------|------|
| 住所 | 郵便番号行を起点に最大3行を自動結合 |
| 会社名 | 社名が2行に分かれている場合に直後1行を結合 |

## ❓ よくある質問

**Q. 白抜き文字の名刺が読み取れない**  
A. 前処理の Step 2.5 で自動検出・反転します。全体が暗い名刺は一括反転、上部バンドのみ暗い名刺はその帯だけを反転します。

**Q. ⓣⓕⓜ などのアイコン付き名刺が正しく分類されない**  
A. 丸囲み文字のアイコン接頭辞は自動的に `TEL:` / `FAX:` / `Mobile:` として認識されます。

**Q. 「電話 045 306-8208」のようにスペース区切りの番号が読み取れない**  
A. `normalizePhoneLine` 関数がスペース区切りをハイフン区切りに自動変換します。

**Q. 「マネージャー　秋山和輝」のように役職と名前が同じ行にある**  
A. PASS 3 の同一行分離処理が自動的に役職と氏名に分割します。

**Q. 「海野花子 Hanako Umino」のように日本語名と英語名が同じ行にある**  
A. PASS 3 の日英名分離処理が自動的に日本語名を氏名フィールドに採用します。

**Q. 「認証:JIS Q 9100:2016」などが分類に混入する**  
A. PASS 4 のノイズ除去で `ISO` / `JIS` / `認証:` で始まる行は自動除去します。

**Q. 「六丁目」などの漢数字住所が正しく処理されない**  
A. OCR 誤認識補正で漢数字の丁目（一〜九丁目）を自動的にアラビア数字に変換します。

**Q. 縦書き名刺の精度が悪い**  
A. 向き選択で「縦向き」を選ぶと `jpn_vert` モデルが追加ロードされ、前処理で 90° 回転してから横書き OCR もかけます。

**Q. データはどこに保存されますか？**  
A. ブラウザの `localStorage` に保存されます。定期的に CSV または Excel でエクスポートすることをお勧めします。
