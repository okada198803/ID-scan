# 📇 名刺スキャナー

> ブラウザだけで動く、完全無料の名刺データ化ツール  
> APIキー不要 / インストール不要 / 外部サーバーへの送信なし

---

## 📌 概要

名刺をカメラ撮影またはファイルアップロードで読み込み、OCR（文字認識）によって情報を自動抽出・分類し、Excel / CSV へ出力できる Web アプリです。  
単一の HTML ファイルで完結しており、ブラウザで開くだけで即使用できます。

---

## ✨ 主な機能

| 機能 | 詳細 |
|------|------|
| **画像入力** | ファイル選択 / ドラッグ＆ドロップ / カメラ撮影（スマートフォン対応） |
| **トリミング** | 読み込み後に名刺部分を切り出し（回転・自由選択対応） |
| **OCR** | Tesseract.js による日本語・英語の混在テキスト認識 |
| **スペース除去** | OCR起因の文字間スペースを自動クリーニング |
| **自動分類** | ルールベースエンジンで8種のフィールドに仕分け |
| **手動修正** | 分類結果を保存前に各フィールドごとに編集可能 |
| **一覧管理** | 登録済み名刺の検索・削除・OCR原文表示 |
| **出力** | Excel（.xlsx）/ CSV（.csv）/ 印刷 の3形式に対応 |

---

## 🚀 使い方

```
1. meishi_scanner.html をダウンロードする
2. ファイルをダブルクリックしてブラウザで開く
3. 名刺画像を読み込む（ファイル選択 or カメラ撮影）
4. トリミングで名刺部分だけを切り出す
5. 「トリミングして OCR へ」をクリック
6. 分類結果を確認・必要なら修正する
7. 「保存する」で名刺リストに追加
8. 右パネルから Excel / CSV へエクスポート
```

### ステップ詳細

#### ① 名刺を読み込む
- **ファイル選択**：JPG / PNG / WebP に対応
- **ドラッグ＆ドロップ**：ブラウザ画面に直接ドロップ
- **カメラ撮影**：スマートフォンのリアカメラ優先で起動。画面内のガイド枠に名刺を合わせて撮影

#### ② トリミング
読み込み直後にトリミング画面が表示されます。名刺以外の背景・余白を除いてからOCRにかけることで認識精度が向上します。

| 操作 | 説明 |
|------|------|
| ドラッグ | 切り出し範囲を移動 |
| 左右回転ボタン | 90° 単位で回転 |
| リセット | 元の状態に戻す |

#### ③ OCR（文字認識）
Tesseract.js が日本語・英語の両モデルでテキストを認識します。進捗バーで処理状況を確認できます。

#### ④ 分類結果を確認・修正
認識されたテキストが8種のフィールドに自動分類されます。  
各フィールドは直接編集可能で、複数の値は改行で区切って入力できます。

---

## 🧠 自動分類ルール

OCR後のテキストを **強ルール → 弱ルール** の2パスで処理し、下記の基準で各フィールドに振り分けます。

| フィールド | 判定基準 |
|-----------|---------|
| **会社名** | 株式会社 / 有限会社 / 合同会社 / Inc. / Ltd. / Holdings 等の法人格キーワード |
| **氏名** | 漢字2〜5文字（単独またはスペース区切り姓名）/ 英語 Firstname Lastname 形式 |
| **役職** | 部長・課長・社長・CEO・Director・Manager 等50種以上のキーワード |
| **電話番号** | `0X-XXXX-XXXX`形式 / 携帯（090/080/070）/ フリーダイヤル / `+81` 国際プレフィックス |
| **メールアドレス** | `@` を含み `.jp` / `.com` / `.net` 等の有効なドメインが続く文字列 |
| **住所** | `〒XXX-XXXX` 郵便番号 / 都道府県名 / 市区町村・丁目・番地・ビル・号室 等 |
| **その他** | 上記いずれにも一致しない行（Web サイト・SNS アカウント・キャッチコピー等） |
| **登録日時** | 保存時に自動付与（手動入力不要） |

---

## 🔧 スペース自動除去

Tesseract.js は文字間に不要なスペースを挿入することがあります。以下のクリーニングを自動実行します。

```
株 式 会 社   →   株式会社          （漢字間スペース除去）
山 田 太 郎   →   山田太郎          （漢字間スペース除去）
user @ mail . jp   →   user@mail.jp    （メールアドレス正規化）
03 - 1234 - 5678   →   03-1234-5678    （電話番号正規化）
〒 123 - 4567   →   〒123-4567        （郵便番号正規化）
```

処理後のテキストは「OCR 生テキスト」欄で確認・手動修正できます。

## 📤 出力形式

### Excel（.xlsx）
- ヘッダー行に **オートフィルター** を自動設定
- 各列の **列幅を最適化**
- **概要シート**（エクスポート日時・件数）を自動追加

### CSV（.csv）
- Excel・Google スプレッドシートどちらでも開ける
- 文字化け対策の **BOM 付き UTF-8** 形式

### 印刷
- ブラウザの印刷ダイアログを開く
- A4 縦向きの一覧表として出力

### 出力列一覧

| 列名 | 内容 |
|------|------|
| 登録日時 | 保存した日時（自動） |
| 会社名 | 法人名・組織名 |
| 氏名 | 担当者氏名 |
| 役職 | 肩書き・ポジション |
| 電話番号1 | メインの電話番号 |
| 電話番号2 | サブの電話番号 |
| メールアドレス1 | メインのメールアドレス |
| メールアドレス2 | サブのメールアドレス |
| 住所 | 勤務先住所 |
| その他 | Web サイト・SNS・備考等 |

## 🔒 プライバシー・セキュリティ

- **すべての処理はブラウザ内で完結**します
- 画像・テキスト・名刺データは**外部サーバーへ一切送信されません**
- 名刺データは**ブラウザの `localStorage`** に保存されます（ブラウザのデータを消去すると削除されます）
- API キーやアカウント登録は**不要**です

## ❓ よくある質問

**Q. OCR の認識精度が低い場合は？**  
A. トリミングで名刺部分だけを切り出すと精度が改善します。また、OCR 生テキスト欄を直接編集して「再分類」ボタンで再処理できます。

**Q. 縦書き名刺は読み取れますか？**  
A. Tesseract.js は縦書きの認識精度が低い場合があります。読み取り後に各フィールドを手動修正してご利用ください。

**Q. データはどこに保存されますか？**  
A. ブラウザの `localStorage` に保存されます。別のブラウザやデバイスからはアクセスできません。定期的に CSV または Excel でエクスポートしてバックアップすることをお勧めします。

**Q. 一度に複数の名刺を登録できますか？**  
A. 1枚ずつ処理する仕様です。登録後は同じ操作を繰り返して複数の名刺を蓄積できます。
